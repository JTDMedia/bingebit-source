BingeBit API Documentation

Welcome to the official API documentation for BingeBit. This API allows you to access and manage movies with ease, including retrieving random films, searching by name, genre, or description, and requesting new film indexing.

API Endpoints

The BingeBit API includes the following endpoints:

  • /list?amount=x: Returns x random films.
  • /find?query=x: Searches for films by name, genre, or description matching x.
  • /request?query=x: Requests indexing for the film x.

API Authentication

To use the API, you need an API key. You can request your key at /request-key.

You can send up to 30 requests per minute. Make sure to monitor your usage to avoid hitting the rate limit.

Request Format

All requests must be made with the appropriate endpoint and the required query parameters. All responses will be returned as JSON.

Example Request

Here’s an example of how to send a request to the API using JavaScript (fetch):


fetch('https://api.bingebit.online/list?amount=5', {
    method: 'GET',
    headers: {
        'Authorization': 'Bearer YOUR_API_KEY'
    }
})
.then(response => response.json())
.then(data => {
    console.log(data);  // The JSON response containing random films
});

Example Responses

The response will be in JSON format. Here’s an example response for the /list endpoint:


{
    "films": [
        { "title": "Film 1", "genre": "Action", "description": "A thrilling action movie." },
        { "title": "Film 2", "genre": "Comedy", "description": "A hilarious comedy." }
    ]
}
